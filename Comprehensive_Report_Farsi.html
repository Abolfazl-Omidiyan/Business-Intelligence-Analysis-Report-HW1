  4. فرضیات آماری: فرض استقلال مشاهدات نقض می‌شود

  2. تورش انتخابی: اگر محصولات گران‌قیمت یا ارزان‌قیمت سیستماتیک ردیف ناقص باشند، توزیع قیمت اریب می‌شود
  3. کاهش نمونه: اندازه نمونه مؤثر کاهش می‌یابد و قدرت آماری کم می‌شود
  4. راه‌حل: استفاده از Imputation (جایگزینی با میانه‌ی دسته) یا حذف متحفظانه

د) اقدامات انجام‌شده:

  • کوئری‌های تکراری حذف شدند (حفظ اولین ردیف)
  • ستون‌های ID برای شناسایی ردیف‌های ناقص بررسی شدند
  • داده‌های ناقص برای تحلیل‌های خاص مجزا رفتار شدند
  • دیتاست‌های پاکیزه‌شده برای تحلیل‌های بعدی ایجاد شدند

نام Bin بازه توضیح مثال

  • استراتژی‌های متفاوت برای هر Bin

✓ نتیجه‌گیری

Binning کوئری‌ها نشان می‌دهد که کاربران از جستجوهای خاص تا عمومی جستجو می‌کنند. این اطلاعات برای بهبود تجربه جستجو و رتبه‌بندی نتایج حائز اهمیت است.

الف) الگوهای مشاهده‌شده:

  1. توزیع شدیداً راست‌چوله (Right-skewed):
    • اکثریت محصولات (>80%) فقط 1 بار بازیابی شده‌اند
    • دنباله طولانی به سمت بازیابی‌های بالاتر (تا 12)
    • میانگین (1.42) > میانه (1.0) نشانه چولگی است
  2. الگوی Long-tail:
    • چند محصول محبوب تمام بازیابی‌ها را تشکیل می‌دهند
    • مثال: محصول 2085 (کتاب) تنهایی 12 بازیابی دارد
    • قانون Zipf: f(k) = 1/k^α (توزیع Power-law)
  3. ویژگی‌های Boxplot:
    • جعبه (Box): نشان‌دهنده 50% داخلی (Q1 تا Q3)
    • خط داخل جعبه: میانه (Q2)
    • سبیل‌ها (Whiskers): تا مقدار معمول (±1.5 IQR)
    • نقاط قرمز: داده‌های پرت (خارج از سبیل‌ها)
  4. ویژگی‌های Histogram:
    • ارتفاع ستون‌ها: تعداد محصولات با آن بازیابی
    • شکل: شدیداً نمایان در سمت چپ (بازیابی کم)
    • دم طولانی: تعداد کمی محصول با بازیابی زیاد
    • خط قرمز: میانگین | خط طلایی: میانه

ب) کاربردهای عملی:

  • شناسایی محصولات محبوب برای تبلیغات
  • شناسایی محصولات Unknown برای Recommendation
  • تنظیم استراتژی موجودی‌سازی
  • بهینه‌سازی سرمایه‌گذاری بازاریابی

✓ نتیجه‌گیری

توزیع بازیابی‌های محصول کاملاً راست‌چوله است و از قانون Zipf پیروی می‌کند. این الگو در سیستم‌های تجارت الکترونیکی عمومی است و بر استراتژی‌های تجاری تأثیر زیادی دارد.

✓ نتایج آزمون:

الف) تفسیر QQ Plot:

  • خط قرمز: آنچه انتظار می‌رفت اگر داده نرمال بود
  • نقاط آبی: داده‌های واقعی (تکرار کوئری‌ها)
  • انحراف شدید: نقاط خیلی دور از خط قرمز
  • نتیجه: داده نرمال نیست!

ب) آزمون آماری Shapiro-Wilk:

  • فرضیه صفر (H₀): داده از توزیع نرمال پیروی می‌کند
  • P-value < 0.0001: خیلی کمتر از سطح معنا‌داری 0.05
  • نتیجه: فرضیه صفر کاملاً رد می‌شود
  • اطمینان: 99.99% که داده نرمال نیست

ج) ویژگی‌های توزیع واقعی:

  1. دم‌های سنگین (Heavy tails): مقادیر شدید بیشتر از انتظار
  2. چولگی مثبت: دم طولانی به سمت راست
  3. کورتوز بالا: اوج تیز‌تر و دم‌های ضخیم‌تر
  4. Power-law یا Log-normal: مدل‌های بهتری برای این داده

د) تأثیرات عملی:

آزمون فرض وضعیت راه‌حل
Student's t-test نرمال ❌ نمی‌شود Mann-Whitney U
ANOVA نرمال ❌ نمی‌شود Kruskal-Wallis
Bootstrap ندارد ✓ می‌شود استفاده بلاتکلیف

ه) توزیع‌های بهتر برای این داده:

  • Power-law: f(x) = x^(-α)
  • Pareto: برای مدل‌سازی بالای 80/20
  • Log-normal: اگر ln(X) نرمال باشد
  • Exponential: برای رویدادهای نادر

✓ نتیجه‌گیری و توصیات

  • نتیجه: داده‌های کوئری از توزیع نرمال پیروی نمی‌کنند
  • علت: Long-tail و Power-law pattern در تجارت الکترونیکی عام است
  • توصیه: استفاده از آزمون‌های ناپارامتری و روش‌های Robust
  • نقطه‌ای: QQ Plot و Shapiro-Wilk نتایج یکسانی دارند

فرضیه صفر (H₀): عضویت در دسته غالب و داشتن کلیک مستقل هستند فرضیه جایگزین (H₁): بین آن‌ها رابطه وجود دارد سطح معناداری: α = 0.05 نتایج: χ² statistic = 0.00 (خیلی کوچک) p-value = 1.0 (خیلی بالا) df (درجه آزادی) = 1 تصمیم: p-value > α → فرضیه صفر رد نمی‌شود

د) تفسیر نتایج:

  • p-value = 1.0: بسیار غیر‌منطقی! هیچ ارتباطی وجود ندارد
  • χ² ≈ 0: تقریباً هیچ اختلافی بین گروه‌ها نیست
  • نتیجه: عضویت در دسته غالب **کاملاً** از کلیک مستقل است
  • معنی: دسته غالب پیش‌بینی‌کننده خوبی برای کلیک نیست

🎯 نتیجه‌گیری جامع پروژه

این پروژه تحلیل جامعی از 5000 کوئری و 4999 محصول در یک سیستم تجارت الکترونیکی ارائه داد. مسائل کیفی داده (1765 کوئری تکراری و 714 قیمت ناقص) شناسایی شدند. با استفاده از روش IQR، 833 داده پرت (16.7%) حذف شد تا دیتاست پاکیزه‌شده برای تحلیل‌های بیشتر آماده شود.

تحلیل آماری نشان داد که داده‌ها از توزیع نرمال پیروی نمی‌کنند (p < 0.0001) بلکه از الگوی Power-law و Zipf's Law پیروی می‌کنند. این الگو در سیستم‌های تجارت الکترونیکی عام است و 80/20 rule را نشان می‌دهد.

رابطه ضعیفی (r = 0.33) بین بازیابی‌ها و کلیک‌ها مشاهده شد، که نشان می‌دهد کیفیت محصول، رتبه‌بندی، و عوامل UX از تنها بازیابی مهم‌تر هستند. کتاب‌ها بیشترین CTR (92%) را داشتند در حالی‌که دسته‌های دیگر برای بهبود نیاز دارند.

نهایتاً، آزمون مربع کای نشان داد که عضویت در دسته غالب هیچ پیش‌بینی‌کننده معنادار برای کلیک نیست (p = 1.0)، که تأکید می‌کند بر اهمیت فاکتورهای دیگر در رتبه‌بندی نتایج.
